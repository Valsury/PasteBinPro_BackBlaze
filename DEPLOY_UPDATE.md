## 🚀 Автоматический деплой (Рекомендуется)

Если настроен **автоматический деплой** с GitHub:

### 1. Проверьте настройки
1. Откройте [Render Dashboard](https://dashboard.render.com/)
2. Найдите ваш сервис `pastebin-pro`
3. Зайдите в **Settings** → **Build & Deploy**
4. Убедитесь что:
   - **Auto-Deploy** = `Yes`
   - **Branch** = `main`

### 2. Дождитесь автодеплоя
- Render автоматически обнаружит новый коммит
- Деплой начнется автоматически (5-10 минут)
- Следите за прогрессом во вкладке **Events**

### 3. Проверьте логи
Перейдите во вкладку **Logs** и убедитесь что:
```
==> Cloning from GitHub
==> Installing dependencies
==> Running build command
==> Build successful
==> Starting service
```

## 🔧 Ручной деплой

Если автодеплой **не настроен** или хотите запустить вручную:

### Способ 1: Manual Deploy
1. [Render Dashboard](https://dashboard.render.com/)
2. Выберите сервис `pastebin-pro`
3. Нажмите **Manual Deploy** → **Deploy latest commit**
4. Подождите завершения (5-10 минут)

### Способ 2: Clear Cache & Deploy
Если возникли проблемы с кешем:
1. **Manual Deploy** → **Clear build cache & deploy**
2. Это пересоберет все с нуля

## 📊 Процесс деплоя

Render выполнит следующие шаги:

```bash
# 1. Клонирование репозитория
==> Cloning from https://github.com/Valsury/PasteBinPro_BackBlaze

# 2. Установка зависимостей
==> pip install -r requirements.txt

# 3. Инициализация БД
==> python -c "from app import app, db; app.app_context().push(); db.create_all()"

# 4. Применение миграций
==> python apply_timezone_migration.py

# 5. Запуск приложения
==> gunicorn app:app --bind 0.0.0.0:$PORT --workers 2 --timeout 120
```

## ⚠️ Возможные проблемы

### Проблема 1: Страницы показывают 404
**Причина**: Старый кеш  
**Решение**:
1. Manual Deploy → Clear build cache & deploy
2. Дождитесь полной пересборки

### Проблема 2: Ошибка при импорте модулей
**Причина**: Не установлены зависимости  
**Решение**: Проверьте логи, убедитесь что `pip install` прошел успешно

### Проблема 3: База данных не работает
**Причина**: Переменная `DATABASE_URL` не установлена  
**Решение**:
1. Settings → Environment
2. Убедитесь что `DATABASE_URL` указывает на вашу PostgreSQL базу

### Проблема 4: Приложение долго загружается
**Причина**: Холодный старт (бесплатный план)  
**Решение**: Это нормально. На бесплатном плане первый запрос после простоя занимает 30-60 секунд
